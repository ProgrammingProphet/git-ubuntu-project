Virtual Machine Setup with Ubuntu & Git

Create and configure a VM on Azure or AWS, install Git, create a repository and initialize it locally, then clone any GitHub repo. Built for beginners, runs as a single index.html.

Part A — Create the VM

Choose your cloud. Steps are nearly identical. Use the tabs below.

Resource group & VM

Azure Portal → Virtual machines → Create. Pick a subscription & resource group, name it (e.g., vm-ubuntu-git), Region close to you, Image: Ubuntu 22.04 LTS, Size: B1s or B2s.

Authentication & networking

Authentication: SSH public key. Generate a new key in the portal or upload your ~/.ssh/id_rsa.pub. Networking: ensure a public IP is attached and allow inbound SSH (22) and HTTP (80).

Launch & connect via SSH

ssh -i ~/.ssh/yourkey.pem azureuser@YOUR_PUBLIC_IP

Launch EC2 (Amazon Linux or Ubuntu)

EC2 Console → Launch instance. Name: vm-ubuntu-git. AMI: Ubuntu 22.04 (or Amazon Linux). Type: t2.micro. Enable Auto-assign Public IP.

Inbound rules: SSH (22) from your IP; HTTP (80) from 0.0.0.0/0. If custom VPC, ensure route to IGW.

Part B — Install Git & Configure

Install Git (Ubuntu)

sudo apt update -y
sudo apt install git -y
git --version

Set your identity

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
# verify
git config --global --list

Generate SSH key (recommended)

ssh-keygen -t ed25519 -C "you@example.com"
# press Enter to accept defaults
cat ~/.ssh/id_ed25519.pub

Copy the printed key and add it in GitHub → Settings → SSH and GPG keys → New SSH key.

Create a new GitHub repo

  1. Open github.com, click New repository
  2. Name: vm-git-practical, Public/Private as you like
  3. Do not tick "Initialize with README" (to practice init locally)
Add screenshot of GitHub new repo page

Initialize a local repo & push

mkdir vm-git-practical && cd vm-git-practical
echo "# VM + Git Practical" > README.md
git init
git add .
git commit -m "Initial commit"

# Replace USER and REPO
# Using SSH (recommended)
git remote add origin git@github.com:USER/vm-git-practical.git
# or HTTPS
# git remote add origin https://github.com/USER/vm-git-practical.git

git branch -M main
git push -u origin main

Clone any repository (as per task)

# Example: clone a sample repo
cd ~
git clone https://github.com/git/git.git
# or using SSH (if your GitHub account has access)
# git clone git@github.com:USER/REPO.git

Part C — Verify & Deploy a Test Page

Serve a basic web page (Apache)

sudo apt update -y || sudo yum update -y
# Ubuntu
sudo apt install apache2 -y && sudo systemctl enable --now apache2
# Amazon Linux
# sudo yum install httpd -y && sudo systemctl enable --now httpd

echo "<h1>Hello from my VM</h1>" | sudo tee /var/www/html/index.html
# Ubuntu path is the same; service name differs (apache2 vs httpd)

Ensure your NSG/SG allows HTTP (80) from 0.0.0.0/0 and the subnet routes to the Internet Gateway (AWS) or has a public IP + UDR to Internet (Azure).

Checklist

VM
VM running (Ubuntu 22.04+). You can SSH into it.
NET
Public IP attached and inbound rules allow 22 & 80.
GIT
Git installed; git --version works; user.name & user.email set.
GH
Repo created on GitHub; pushed initial commit from VM.
CLONE
Cloned a public repo locally using HTTPS or SSH.

Troubleshooting

SSH fails

  • Wrong username: ubuntu (Ubuntu), ec2-user (Amazon Linux), azureuser (Azure default)
  • Permissions on key: chmod 600 ~/.ssh/yourkey.pem
  • Security rules must allow SSH (22) from your IP

Website not loading

  • Service running? systemctl status apache2 or systemctl status httpd
  • Port open? ss -ltnp '( sport = :80 )'
  • Public IP + route to Internet Gateway (AWS) or correct UDR (Azure)

Git auth issues

  • For SSH, add ~/.ssh/id_ed25519.pub to GitHub → SSH keys
  • Test: ssh -T git@github.com (type yes on first connect)
  • For HTTPS, use a GitHub Personal Access Token (classic) as password

Reset remotes

git remote -v
git remote remove origin
# SSH
git remote add origin git@github.com:USER/REPO.git
# HTTPS
# git remote add origin https://github.com/USER/REPO.git
